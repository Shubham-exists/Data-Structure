# Find Minimum in Rotated Sorted Array

**Problem Link**:
LeetCode – Find Minimum in Rotated Sorted Array

## Problem Statement

You are given a rotated sorted array of distinct integers. A rotated array means that the original sorted array was shifted at some pivot. Your task is to find the smallest element in the array.

The solution must run in O(log n) time.

## Example

Input: nums = [3,4,5,1,2]
Output: 1

## Approach

### **1. Brute Force Approach**

  • Scan through the entire array.
  • Track the minimum element manually.
  • Very straightforward but inefficient for large inputs.

Time Complexity: O(n)
Space Complexity: O(1)

### **2. Binary Search Approach (Optimal)**

This is the required method to achieve O(log n) complexity.

Key observations:

  • The array is partially sorted and partially rotated.
  • One side of the array will always be sorted.
  • The minimum will always lie in the unsorted portion.
  • Compare the middle element with the last element:
    • If mid > right, the pivot is on the right half.
    • Otherwise, it’s on the left half.

This selective elimination reduces the search space logarithmically.

Time Complexity: O(log n)
Space Complexity: O(1)

## Time & Space Complexity Summary

  • Time Complexity: O(log n)
  • Space Complexity: O(1)
